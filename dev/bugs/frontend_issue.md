# 前端實作問題與解決方案

## 1. BaseButton 元件 Class 衝突 (2026-05-28)

### 問題描述

法律規範彈出視窗（Legal Modal）中的「不同意 (Decline)」按鈕，即使加上了 `text-white`，看起來仍然灰灰暗暗的。嘗試透過簡單的字串替換來修復，卻始終無法呈現預期的「純白」視覺效果。

### 根本原因分析

* **Class 優先級衝突：** `BaseButton.vue` 元件的基礎樣式中寫死了 `font-medium`，但 ghost 變體（variant）卻試圖套用 `font-bold`。在 Tailwind 的 JIT 模式或 CSS 層疊規則（cascade）中，這兩個 class 發生了競爭，通常最終渲染出來的是較細的字體。
* **視覺比重感知：** 在暗色或灰色的彈出視窗背景下，相較於粗體 (`font-bold`)，較細的白色字體 (`font-medium`) 筆畫覆蓋面積較小，導致肉眼看起來不夠「白」，反而偏向「灰色」。
* **架構耦合：** 基礎元件過於「強勢」（帶有自己的主觀視覺風格），這違反了原子設計（Atomic Design）的原則。在該原則中，變體（variants）應該擁有對視覺識別的完整控制權。

### 解決方案與預防措施

* **視覺去中心化：** 從基礎樣式字串中，移除帶有視覺主觀意見的 class（例如字體粗細、特定顏色）。
* **明確的變體定義：** 確保每個變體都明確定義其與排版無關的視覺屬性（顏色、字重、透明度），以避免繼承或衝突問題。
* **重構策略：** 基礎樣式（Base classes）專注於處理排版（如 `flex`, `px`, `py`, `rounded`）；而變體樣式（variantClasses）則專門處理美觀層面（如 `text-color`, `bg-color`, `font-weight`）。
